Case Study: MyFlix App

Overview

The MyFlix App is a web application developed using the MERN stack. It allows users to create an account, view and search a list of movies, get detailed information on each movie, add movies to their "Favorites," and manage their profile details. This project was part of the Full-Stack Web Development course at CareerFoundry, where I built the application from scratch, integrating server-side and client-side components into a cohesive and functional product.

Purpose and Context

The MyFlix App was conceived as a learning project within the context of my Full-Stack Web Development course at CareerFoundry. The primary goal was to gain hands-on experience with the MERN stack by building a fully functional web application. This project served as an opportunity to practice and master in-demand development skills, including database management, API creation, and front-end development, while providing a tangible product to showcase in my portfolio.

Objective

The primary objective of this project was to develop proficiency in the MERN stack by creating a web application from the ground up. This involved mastering MongoDB for database management, Express and Node.js for server-side development, and React for building the client-side interface. The project culminated in the creation of the MyFlix App, which not only met the course requirements but also provided me with a comprehensive understanding of full-stack development.

Approach

The MERN stack comprises four core technologies:

  • MongoDB: A NoSQL document database ideal for handling JSON data.
  • Express: A flexible web application framework for Node.js.
  • React: A powerful JavaScript library for building user interfaces.
  • Node.js: A JavaScript runtime environment that enables server-side scripting.

I began by developing each component individually, starting with the database, followed by the server-side API, and finally, the client-side interface. Once each part was functional, I integrated them to create the final product, ensuring seamless interaction between the server and client sides.

Database

For the database, I used MongoDB, a non-relational database optimized for storing and retrieving JSON data. I created collections for "movies" and "users," each with the necessary attributes, using Mongo Shell. Mongo Atlas was employed for managing and monitoring the database, providing an intuitive interface to visualize data and perform database operations.

Server-Side Development

The server-side component was developed using Express and Node.js. I built a RESTful API to make the JSON data accessible, utilizing HTTP methods such as GET and POST for data retrieval and submission. CRUD (Create, Read, Update, Delete) operations were implemented to interact with the database. For security, I incorporated HTTP and JWT authentication, ensuring that only authenticated users could access specific API endpoints. The API's functionality was thoroughly tested using Postman, allowing me to verify the accuracy of the HTTP methods and authentication processes.

Client-Side Development

After completing the server-side components, I developed the client-side interface using React. The interface includes several key features:

  • Registration and Login: Components that allow users to create an account and log in securely.
  • Main View: A comprehensive list of movies fetched from the database.
  • Movie Cards: Individual cards displaying detailed information about each movie.
  • Favorites: Functionality enabling users to add movies to their "Favorites" list.
  • Profile View: A section where users can view and update their details, as well as manage their favorite movies.

To enhance the user experience, I implemented Redux to manage state and handle the search functionality on the main view page, ensuring efficient and responsive filtering of movies.

Challenges and Solutions

One of the significant challenges I faced was with React, particularly in handling authentication tokens and the users/username endpoint. Another challenge involved updating the profile view automatically when user information changed. These issues required extensive troubleshooting, guidance from my tutor and mentor, and collaborative problem-solving through peer-programming sessions. Overcoming these obstacles not only deepened my understanding of React but also improved my problem-solving skills.

Final Thoughts

Developing the MyFlix App was a rewarding experience that significantly advanced my skills in full-stack development. The process of building this application from the ground up—and overcoming the inevitable challenges—has given me a deep familiarity with the MERN stack. While this project required considerable time and effort, it has equipped me with the knowledge and confidence to tackle similar projects more efficiently in the future.
